# AdventureWorks scaffolded with EF Core 9

This project contains `AdventureWorksDbContext` and 90 entity classes
generated by `dotnet ef dbcontext scaffold` against the
**AdventureWorks2022** sample database, using **EF Core 9** tooling.

See `../AdventureWorks-EF8/SCAFFOLDING.md` for the full rationale,
the flag choices (omits `--use-database-names` and `--data-annotations`
to match the existing EF7 style), the list of skipped tables
(`Document`, `ProductDocument`), and the two architectural mismatches
that block the test-csproj rewire (record/class diff +
SQL-Server-only extensions). All of that is identical between EF8,
EF9, and EF10.

## Scaffold command actually run

From the repo root, with `dotnet-ef 9.x` installed globally:

```bash
dotnet ef dbcontext scaffold \
  "Server=CWOLFGAN-03\SQLEXPRESS;Database=AdventureWorks2022;Integrated Security=SSPI;TrustServerCertificate=True" \
  Microsoft.EntityFrameworkCore.SqlServer \
  --project extra-projects/AdventureWorks-EF9/AdventureWorks-EF9.csproj \
  --output-dir Models/Generated \
  --context AdventureWorksDbContext \
  --context-namespace AdventureWorks.Models \
  --namespace AdventureWorks.Models \
  --no-onconfiguring \
  --force
```
